# Vue View 页面编码约定

## 页面结构
```vue
<template>
  <div class="page-shell">
    <!-- 页头：标题 + 操作按钮 -->
    <div class="page-head">
      <div>
        <h2 class="page-title">模块名称</h2>
        <div class="page-subtitle">一句话描述</div>
      </div>
      <el-button v-if="canManage" type="primary" @click="onOpenCreate">新增</el-button>
    </div>

    <!-- 统计卡片行 -->
    <el-row :gutter="16" class="stats-row">
      <el-col :span="5"><el-card shadow="never" class="stat-card">...</el-card></el-col>
    </el-row>

    <!-- 筛选区域 -->
    <AppFilterCard>
      <el-form :model="query" inline class="filter-form" @submit.prevent="onSearch">
        ...
        <el-form-item class="filter-actions">
          <el-button type="primary" @click="onSearch">查询</el-button>
          <el-button @click="onReset">重置</el-button>
        </el-form-item>
      </el-form>
    </AppFilterCard>

    <!-- 数据表格 -->
    <AppTableCard>
      <el-table :data="tableData" v-loading="loading" stripe max-height="520">
        ...
      </el-table>
    </AppTableCard>

    <!-- 分页 -->
    <el-pagination ... />
  </div>
</template>
```

## 脚本约定
- 使用 `<script setup lang="ts">` Composition API
- 类型从 `@/types/{module}` 导入
- API 函数从 `@/api/modules/{module}` 导入
- 权限判断：`const canManage = computed(() => ...)` / `canApprove`

## 响应式断点
| 断点 | 用途 |
|------|------|
| ≤992px | 隐藏侧边栏，表格列折叠 |
| ≤768px | 筛选表单垂直排列 |
| ≤576px | 移动端优化，统计卡片堆叠 |

## UI 组件
- 表格：`el-table` + `el-table-column`（固定操作列 `fixed="right"`）
- 表单：`el-form` + `el-form-item`（行内筛选用 `inline`）
- 弹窗：`el-dialog` + `v-model` 控制
- 日期：`el-date-picker type="daterange"` + `value-format="YYYY-MM-DD"`
- 下拉：`el-select filterable clearable`
- 加载：`v-loading` 指令

## 硬性规则
1. **所有页面用 `.page-shell` 包裹**
2. **统计卡片用 `el-card shadow="never" class="stat-card"`**
3. **筛选区域用 `<AppFilterCard>` 组件**
4. **表格区域用 `<AppTableCard>` 组件**
5. **权限按钮必须用 `v-if="canManage"` 等条件守卫**
6. **API 错误用 `ElMessage.error()` 提示**
7. **导出下载用 blob + `URL.createObjectURL` 模式**

## 标杆文件
参考 `workhours/WorkHoursView.vue` 了解完整模式。

## 禁止
- 不要使用 Options API
- 不要在 template 中写复杂逻辑，提取为 computed 或方法
- 不要硬编码角色名，统一走 `canManage` / `canApprove` 计算属性
