# API 客户端编码约定

## 文件组织
- 一个后端模块对应一个 `.ts` 文件，放在 `modules/` 子目录
- 文件名与后端模块对应：`workhours.ts` ↔ `WorkHoursController`

## 导入模式
```typescript
import request from '../request'
import type { ApiResponse, PagedResult } from '../../types/project'
import type { XxxItem, XxxQuery, XxxUpsert, XxxSummary } from '../../types/xxx'
```

## CRUD 函数命名
| 操作 | 命名模式 | HTTP 方法 |
|------|----------|-----------|
| 汇总统计 | `fetchXxxSummary()` | GET |
| 分页查询 | `fetchXxxList(params: XxxQuery)` | GET |
| 单条查询 | `fetchXxxById(id: number)` | GET |
| 新增 | `createXxx(data: XxxUpsert)` | POST |
| 修改 | `updateXxx(id: number, data: XxxUpsert)` | PUT |
| 删除 | `deleteXxx(id: number)` | DELETE |
| 导出 | `exportXxx(params?: Partial<XxxQuery>)` | GET + blob |

## 函数模板
```typescript
export function fetchXxxList(params: XxxQuery) {
  return request.get<any, ApiResponse<PagedResult<XxxItem>>>('/xxx', { params })
}

export function exportXxx(params?: Partial<XxxQuery>) {
  return request.get<any, Blob>('/xxx/export', { params, responseType: 'blob' })
}
```

## 硬性规则
1. **使用 `request` 实例**（来自 `../request`），不要直接用 `axios`
2. **泛型响应类型**：普通请求用 `ApiResponse<T>`，分页用 `ApiResponse<PagedResult<T>>`
3. **导出请求**：加 `responseType: 'blob'`，返回类型为 `Blob`
4. **类型定义放 `../../types/` 目录**，不在 API 文件中定义
