# 滚动条与筛选复测清单（2026-03-03）

> 目标：验证“横向滚动可见性”和“统计卡片筛选不叠加旧条件”在高频页面稳定生效。  
> 覆盖页面：项目台账、权限管理、巡检管理（含巡检结果）。

---

## 0. 复测前准备

1. 启动前端并进入系统首页。
2. 浏览器窗口保持常用分辨率（建议 1366×768 或 1920×1080）。
3. 每个页面复测前先点一次“重置”，确保无历史筛选残留。

---

## 1. 项目台账（ProjectList）

### 用例 1：横向滚动条固定可见
- 步骤：进入“项目台账”，不滚动整个页面，只观察表格区域底部。
- 预期：表格内部可直接看到横向滚动条；无需把整页滚到最底。

### 用例 2：卡片/钻取不叠加旧筛选
- 步骤：
  1) 手工筛选：省份=任意、组别=任意；
  2) 点击状态卡片（如“超期未签署”）；
  3) 查看筛选项状态与结果。
- 预期：
  - 冲突筛选项被清空；
  - 列表结果仅按当前卡片条件过滤；
  - 结果数量与卡片口径一致。

### 用例 3：分页回落一致
- 步骤：重置后观察分页大小；切换页面后返回本页再看一次。
- 预期：每页条数默认保持 15。

---

## 2. 权限管理（PersonnelList）

### 用例 1：宽表横向滚动可用
- 步骤：进入“权限管理”，观察“员工编号~组别~操作”等多列宽表。
- 预期：
  - 表格内部横向滚动可直接拖动；
  - 不需要滚动外层页面到底部。

### 用例 2：统计卡片不叠加旧查询
- 步骤：
  1) 先设置查询：姓名/部门/组别任意；
  2) 点击卡片“服务人员”或“实施人员”；
  3) 观察筛选项与结果。
- 预期：
  - 非卡片目标的冲突筛选会被清空；
  - 点击“服务人员”只按角色=服务过滤；
  - 点击“驻场人数”只按驻场=是过滤。

### 用例 3：总数卡片恢复全量口径
- 步骤：先点“服务人员”，再点“总人数”。
- 预期：返回全量数据口径，不残留服务/驻场等限制。

---

## 3. 巡检管理（InspectionPlan）

### A. 巡检计划 Tab

#### 用例 A1：横向滚动条固定可见
- 步骤：进入“巡检管理-巡检计划”，观察表格底部。
- 预期：表格内部横向滚动条可见、可拖动。

#### 用例 A2：状态卡片点击清空冲突筛选
- 步骤：
  1) 设置筛选：省份/产品/组别/巡检人任意；
  2) 点击状态卡片（如“已完成”）；
  3) 查看筛选项和列表数据。
- 预期：
  - 省份/产品/组别/巡检人会清空；
  - 只保留状态筛选；
  - 列表结果与卡片状态一致。

#### 用例 A3：分页默认一致
- 步骤：重置后查看分页大小；刷新页面后再看。
- 预期：默认每页 15。

### B. 巡检结果 Tab

#### 用例 B1：健康等级卡片不叠加旧筛选
- 步骤：
  1) 在“巡检结果”先设医院/产品/巡检人；
  2) 点击“警告”或“严重”卡片；
  3) 观察筛选项与列表。
- 预期：
  - 医院/产品/巡检人被清空；
  - 只按健康等级过滤。

#### 用例 B2：详情弹窗风险表可滚动
- 步骤：打开一条有风险项的“详情”弹窗，观察风险项子表。
- 预期：子表可在内部滚动，横向不被外层裁切。

## 4. 回归通过判定

- 任一页面出现“必须整页滚到底才看到横向滚动条” => **不通过**。
- 任一卡片点击后仍保留冲突筛选（导致口径变小或不准） => **不通过**。
- 分页默认不是 15（在本次修复范围页面） => **不通过**。
