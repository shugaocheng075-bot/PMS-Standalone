# UI 逐页巡检清单（单行标签 / 紧凑排版 / 一致性）

日期：2026-03-03
范围：`pms-web` 全站列表、看板、统计卡片、页签区域

## 一、全局验收标准

- 统计标签保持单行（不换行），空间不足时可横向滚动。
- 统计卡片在同一行内高度观感一致（数字与标题垂直居中）。
- 看板行（kanban）与统计行（stats）采用同一单行策略。
- 页签文本默认不换行，避免换行导致高度跳变。
- 页面整体间距为紧凑风格（页头、筛选、表格、分页行高一致）。

## 二、页面巡检清单（建议按顺序）

### 1) 首页与看板
- DashboardView
- HandoverListView（含 `kanban-row`）

检查项：
- 统计卡片是否保持单行排列；
- 是否出现多行折返；
- 缩小窗口后是否改为横向滚动而非换行；
- 卡片高度与数字基线是否一致。

### 2) 预警与报表
- AlertCenterView
- ContractAlertView
- AnnualReportView

检查项：
- 统计区是否一行展示；
- 文本是否被异常截断（应保留省略或滚动，不应换行撑高）；
- 表格顶部筛选区与统计区间距是否统一。

### 3) 主数据与台账
- HospitalListView
- ProductListView
- PersonnelListView
- ProjectListView
- InspectionPlanView
- MajorDemandView

检查项：
- 统计卡片是否与其他模块视觉一致；
- 表格密度（行高/头部）是否符合紧凑规范；
- 分页器与内容块间距是否一致。

### 4) 其他带页签页面
- 所有含 Element Plus Tabs 的页面

检查项：
- Tab 文本是否保持单行；
- 切换标签时顶部高度是否稳定；
- 小屏下是否出现异常换行导致抖动。

## 三、通过判定

以下条件全部满足即判定通过：
- 无统计标签换行；
- 无卡片高低不齐的明显视觉跳变；
- 无 tabs 换行导致的容器高度抖动；
- 页面紧凑样式在主要模块一致。

## 四、当前代码已落实的统一规则

- `src/style.css`
  - `.stats-row`：单行 + 横向滚动 + 子列最小宽度
  - `.kanban-row`：单行 + 横向滚动 + 子列最小宽度
  - `.el-tabs__item`：默认单行
  - 统计卡片 `.el-card__body`：统一最小高度与垂直居中

- `src/layout/AppLayout.vue`
  - 管理员显示为 `admin`
  - 顶部人员切换下拉已移除

---
如有个别页面仍出现换行，优先在页面容器上补“局部最小宽度”而不破坏全局规则。
