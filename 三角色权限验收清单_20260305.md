# PMS 三角色权限验收清单（2026-03-05）

## 1. 验收目标
- 验证 `operator / supervisor / manager` 在前后端行为一致。
- 验证“可见即可用，不可用即禁止”（UI 按钮与 API 返回一致）。
- 覆盖模块：项目、工时、报修、医院、重大需求、人员权限分配。

## 2. 预置条件
- API：`http://127.0.0.1:5111`（已可访问 `/swagger`）。
- Web 已构建通过（`npm run build`）。
- 准备 3 个账号：
  - `operator`（普通运维）
  - `supervisor`（运维主管）
  - `manager`（经理）
- 准备测试数据：至少 2 家医院、2 个项目，其中 1 个项目不在 operator 负责范围。

## 3. 角色验收矩阵

### 3.1 operator（普通运维）
- 项目：
  - 只能看到自己范围项目。
  - 不能编辑/删除项目。
- 报修：
  - 可新增报修（自己范围内）。
  - 不可编辑/删除报修。
  - 可导出自己范围数据。
- 重大需求：
  - 可新增/评论进度（自己范围内）。
  - 不可批量管理。
- 工时/医院/权限分配：
  - 不能管理。

### 3.2 supervisor（运维主管）
- 项目：
  - 可管理所辖运维人员项目（编辑/批量更新按范围有效）。
- 合同与项目信息：
  - 可查看。
- 权限分配：
  - 不可进行经理级权限分配操作。

### 3.3 manager（经理）
- 人员权限：
  - 可设置角色、上级、医院范围、权限项。
- 业务管理：
  - 可管理工时、项目、报修、医院、合同、重大需求。
- 重大需求：
  - 可批量管理、评论、导出。

## 4. 关键页面检查点（UI）
- 项目列表：`ProjectListView` 编辑/删除按钮仅 `manager/supervisor` 可见可用。
- 人员列表：`PersonnelListView` 权限分配入口仅 `manager` 可用。
- 医院列表：`HospitalListView` 管理按钮仅 `manager` 可用。
- 工时列表：`WorkHoursView` 新增/编辑/删除仅 `manager` 可用。
- 报修列表：`RepairRecordView`
  - 新增：`operator + manager`
  - 编辑/删除：仅 `manager`
- 重大需求：`MajorDemandView`
  - 批量管理：仅 `manager`
  - 评论/进度：`operator + manager`

## 5. 关键 API 检查点（建议用 Swagger 或 Postman）
- 权限分配相关（角色/上级/医院范围/权限保存）：非 `manager` 应拒绝。
- 项目更新/删除/批量更新：`operator` 应拒绝；`supervisor/manager` 按 scope 成功。
- 报修 create/update/delete：
  - create：`operator/manager`（并受 scope 约束）
  - update/delete：仅 `manager`
- 重大需求列表/导出/批量：按角色 + scope 生效，空 scope 不应泄漏数据。

## 6. 判定标准
- UI 与 API 结果一致：
  - UI 不可操作时，API 同样拒绝（防越权）。
  - UI 可操作时，API 在 scope 内成功、scope 外失败/无数据。
- 分页前过滤正确：跨页不应出现越权数据。

## 7. 复测建议顺序
1. 先用 `operator` 全流程走一遍（最容易发现越权）。
2. 再用 `supervisor` 验证“可管理所辖范围”。
3. 最后用 `manager` 验证全权限与分配能力。
4. 每个角色至少抽检 1 条 scope 外数据，确认被拦截。
